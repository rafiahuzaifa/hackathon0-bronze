# SKILL: CEO Monday Morning Briefing

## Goal
Generate a comprehensive, data-driven Monday briefing every Sunday at 22:00. The CEO reads this first thing Monday morning to understand the state of the business and the AI Employee system.

## Data Sources

| Source | Data Extracted |
|--------|---------------|
| `vault/Business_Goals.md` | Q1 targets, KPIs, client list |
| `vault/Done/` | Completed tasks this week (count by category) |
| `vault/Needs_Action/` | Pending inbox count |
| `vault/Pending_Approval/` | Awaiting human decisions |
| `vault/Logs/*.json` | Bot activity, errors, anomalies |
| `uploads/bank/latest.csv` | Revenue, expenses, anomalies |

## Steps

1. **Scan** `vault/Done/` — count tasks completed in last 7 days, group by type
2. **Parse** bank transactions — calculate weekly income, expenses, net
3. **Read** Business_Goals.md — compare actual vs targets for all KPIs
4. **Scan** Logs/*.json — extract: errors, anomalies, high-risk events, bot uptime
5. **Detect bottlenecks** — tasks stuck > 24h, recurring errors, approval backlog
6. **Generate suggestions** — based on patterns (see Suggestion Rules)
7. **Assemble briefing** in the format below
8. **Write** to `vault/Briefings/{YYYY-MM-DD}_Monday_Briefing.md`
9. **Update** `vault/Dashboard.md` with briefing link
10. **Log** completion event

## Suggestion Rules

| Pattern Detected | Suggestion |
|-----------------|-----------|
| Subscription cost > 30% of expenses | "Audit SaaS subscriptions — {list tools}" |
| Revenue < 80% of target | "Follow up with {client} re: overdue invoice" |
| Email backlog > 10 | "Consider expanding auto-reply rules" |
| Bot error rate > 5% | "Review {bot} logs — recurring {error_type}" |
| Approval queue > 5 items | "Clear approval backlog — {n} items waiting" |
| No new leads this week | "Schedule outreach campaign" |

## Rules
- ALWAYS run even if some data sources are missing (use "N/A" for missing data)
- Never include raw PII in briefing (anonymise names)
- Financial figures always in PKR with comma formatting
- Use emoji indicators: 🟢 on-track, 🟡 at-risk, 🔴 critical
- Max briefing length: 800 words (concise executive summary)
- DRY_RUN note at bottom when applicable
